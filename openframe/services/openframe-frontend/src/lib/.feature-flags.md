Centralized feature flags system that controls feature availability across the application based on runtime environment configuration.

## Key Components

- **featureFlags** - Main feature flags object with methods to check feature availability
- **FeatureFlagKey** - Type definition for valid feature flag keys

## Usage Example

```typescript
import { featureFlags } from './feature-flags'

// Check whether a feature is enabled
if (featureFlags.subscription.enabled()) {
  // ...
}
```

The feature flags are determined by the underlying `runtimeEnv` configuration plus server-loaded values, providing a consistent interface for conditional feature rendering throughout the application.

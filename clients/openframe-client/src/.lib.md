The main entry point and service orchestrator for the OpenFrame agent client, providing unified access to all core services and managing their dependencies.

## Key Components

- **ClientConfiguration**: Configuration structs for server, client, metrics, and security settings
- **SessionBus**: Type alias for the Windows SCM SessionChange event channel — `mpsc::UnboundedReceiver<SessionEvent>` on Windows, `()` elsewhere. Lets `Client::new` and `Service::run` keep a single signature across platforms.
- **Client**: Main orchestrator that initializes and manages all services including:
  - Registration and authentication processors
  - NATS connection manager for real-time messaging
  - Tool installation and update listeners
  - Tool run and connection managers
  - Machine heartbeat services
  - Directory and file management services
  - On Windows, an optional `WindowsSessionManager` constructed from `session_bus` (service mode only)

## Usage Example

```rust
use openframe_agent_lib::{Client, ClientConfiguration};

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    // Create a new client instance. `None` means no SCM session bus —
    // used for direct/foreground runs; service mode passes `Some(rx)`.
    let client = Client::new(None)?;
    
    // The client automatically initializes all required services:
    // - HTTP client with proper timeout and TLS settings
    // - Directory manager for file operations
    // - Registration and authentication processors
    // - NATS connection for real-time messaging
    // - Tool installation and update listeners
    // - Machine heartbeat services
    
    // Configuration is loaded from default or environment variables
    let config = ClientConfiguration::default();
    println!("Server URL: {}", config.server.url);
    println!("Update channel: {}", config.client.update_channel);
    
    Ok(())
}
```

The Client struct handles dependency injection and service initialization, supporting both production and development modes through environment variables.

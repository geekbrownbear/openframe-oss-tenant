Windows-only helpers for working with wide-string Win32 APIs and the HKLM autorun registry hive. Centralizes a few small utilities used by `ToolRunManager` and `WindowsSessionManager` so the FFI plumbing isn't duplicated across call sites.

## Key Components

- **`to_wide(s)`** - Converts a Rust `&str` to a null-terminated UTF-16 `Vec<u16>` suitable for `PCWSTR`/`PWSTR` arguments to wide-character Win32 APIs.
- **`wcslen(ptr)`** - Computes the length (in `u16`s) of a null-terminated UTF-16 string pointed to by `ptr`. Caller is responsible for the pointer's lifetime and validity.
- **`build_command_line(command_path, args)`** - Builds a Windows command-line string of the form `"command_path" arg1 "arg with spaces" ...`. The executable path is always quoted; individual args are quoted only when they contain spaces.
- **`register_autorun(value_name, command_path, args)`** - Idempotently writes an entry to the 64-bit HKLM Run key (`Software\Microsoft\Windows\CurrentVersion\Run`) so Windows launches the app at every user's logon. Only writes if the existing value differs. Defensively deletes any stale entry from the WOW6432Node mirror to avoid double-launches.
- **`unregister_autorun(value_name)`** - Removes the entry from both the 64-bit and WOW6432Node Run-key views. Idempotent — does nothing (and returns Ok) if the value is absent.

## Usage Example

```rust
use crate::utils::windows_helpers::{register_autorun, unregister_autorun, build_command_line};

// Build a command line for CreateProcessW / CreateProcessAsUserW.
let cmdline = build_command_line(
    r"C:\Program Files\OpenFrame\openframe-chat.exe",
    &["--background".to_string()],
);
// → `"C:\Program Files\OpenFrame\openframe-chat.exe" --background`

// Register a GuiApp for autostart at every user's logon.
register_autorun(
    "openframe-chat",
    r"C:\ProgramData\OpenFrame\tools\openframe-chat\openframe-chat.exe",
    &["--background".to_string()],
)?;

// Later, on uninstall, remove the autorun entry.
unregister_autorun("openframe-chat")?;
```

The Run-key entry is HKLM (machine-wide), not HKCU, so it applies to every user that logs into the machine — pairing with `WindowsSessionManager`, which observes the launched processes via WTS enumeration and attaches per-session waiters.

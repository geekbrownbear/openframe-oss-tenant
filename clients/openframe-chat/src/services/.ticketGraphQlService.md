A GraphQL service that handles support ticket operations for the chat client: listing and reading tickets with cursor-based pagination, creating new tickets, and managing temporary file attachments for ticket creation.

## Key Components

- **TicketGraphQlService** - Main service class managing GraphQL client lifecycle and authentication
- **ticketGraphQlService** - Exported singleton instance for application-wide use
- **TicketNode / TicketEdge / TicketsConnection** - Ticket data model with cursor-based pagination (`pageInfo`, `edges`)
- **TicketStatusDefinition** - Lifecycle (custom-status) definition, fetched only when the `lifecycle` variable is `true` (driven by the `ticket-statuses` feature flag)
- **CreateTicketInput / UserError / TempAttachment** - Input and result types for mutations
- **GraphQL Documents** - `GET_TICKETS_QUERY`, `GET_TICKET_QUERY`, `CREATE_TICKET_MUTATION`, `CREATE_TEMP_ATTACHMENT_UPLOAD_URL`, `DELETE_TEMP_ATTACHMENT`

## Usage Example

```typescript
import { ticketGraphQlService } from './ticketGraphQlService'

// List open tickets (first page)
const connection = await ticketGraphQlService.getTickets({
  statuses: ['OPEN'],
  limit: 20,
  lifecycle: true, // include statusDefinition when the ticket-statuses flag is on
})
if (connection) {
  for (const edge of connection.edges) {
    console.log(`#${edge.node.ticketNumber}: ${edge.node.title} (${edge.node.status})`)
  }
}

// Read a single ticket
const ticket = await ticketGraphQlService.getTicket('ticket-id', true)

// Create a ticket with an attachment
const upload = await ticketGraphQlService.createTempAttachmentUploadUrl('screenshot.png', 'image/png')
// ...PUT the file to upload.uploadUrl...
const created = await ticketGraphQlService.createTicket({
  title: 'My computer is running slow',
  description: 'Started this morning',
  tempAttachmentIds: [upload.id],
})

// Clean up when done
ticketGraphQlService.dispose()
```

The service automatically handles authentication via `tokenService` (waiting for the token with `ensureTokenReady`) and reuses a cached GraphQL client with refreshed headers per call. Read operations (`getTickets`, `getTicket`, `deleteTempAttachment`) swallow errors and return `null`/`false`, while mutations (`createTicket`, `createTempAttachmentUploadUrl`) throw on transport errors and surface `userErrors` as exceptions.

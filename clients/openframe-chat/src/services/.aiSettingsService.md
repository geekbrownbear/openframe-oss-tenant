A GraphQL service that loads the client assistant's customization for the chat UI. The backend splits this across two collections, fetched together in one request: appearance (display name, avatar, application theme, accent color) from `clientView`, and the configured quick actions from `clientAiConfig`. The result is flattened into a single chat-facing shape.

## Key Components

- **AiSettingsService** - Main service class managing GraphQL client lifecycle and authentication
- **aiSettingsService** - Exported singleton instance for application-wide use
- **AiSettingsResponse** - Flattened chat-facing record (ClientView id, assistant name, avatar, application theme, accent color, quick actions)
- **AiQuickAction** - Interface for a single quick action (`id`, `name` shown as the button label, `instructions` sent into the dialog on click)
- **AI_SETTINGS_QUERY** - Pre-defined `clientView(organizationId)` + `clientAiConfig` query (two root fields in one request)

## Usage Example

```typescript
import { aiSettingsService } from './aiSettingsService'

// Fetch the client assistant's appearance + quick actions (null when neither exists yet)
const settings = await aiSettingsService.fetchAiSettings()
if (settings) {
  console.log(`Assistant: ${settings.assistantName}`)

  // Render configured quick actions
  for (const action of settings.quickActions ?? []) {
    console.log(`${action.name}: ${action.instructions}`)
  }
} else {
  console.log('No settings record yet - fall back to bundled defaults')
}
```

The service automatically handles authentication via `tokenService` (waiting for the token with `ensureTokenReady`), reuses a cached GraphQL client with refreshed headers per call, and throws on transport or GraphQL errors so callers (e.g. `useAiSettingsQuery`) can decide how to fall back.

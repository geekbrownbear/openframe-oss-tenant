A GraphQL service that loads FaeSettings (AI assistant customization) for the chat client: the assistant's display name, avatar, LLM provider and model, application theme, accent color, and the configured quick actions shown in the chat UI.

## Key Components

- **FaeSettingsService** - Main service class managing GraphQL client lifecycle and authentication
- **faeSettingsService** - Exported singleton instance for application-wide use
- **FaeSettingsResponse** - Interface for the full settings record (assistant name, avatar, provider, theme, accent color, quick actions, timestamps)
- **FaeQuickAction** - Interface for a single quick action (`id`, `name` shown as the button label, `instructions` sent into the dialog on click)
- **FAE_SETTINGS_QUERY** - Pre-defined `faeSettings(organizationId)` query fetching the complete settings record

## Usage Example

```typescript
import { faeSettingsService } from './faeSettingsService'

// Fetch the current FaeSettings record (null when none exists yet)
const settings = await faeSettingsService.fetchFaeSettings()
if (settings) {
  console.log(`Assistant: ${settings.assistantName}`)

  // Render configured quick actions
  for (const action of settings.quickActions ?? []) {
    console.log(`${action.name}: ${action.instructions}`)
  }
} else {
  console.log('No settings record yet - fall back to bundled defaults')
}
```

The service automatically handles authentication via `tokenService` (waiting for the token with `ensureTokenReady`), reuses a cached GraphQL client with refreshed headers per call, and throws on transport or GraphQL errors so callers (e.g. `useFaeSettingsQuery`) can decide how to fall back.

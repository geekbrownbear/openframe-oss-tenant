A GraphQL service that fetches frontend feature flags for the chat client, returning the effective enabled/disabled state for an explicitly requested list of flag names.

## Key Components

- **FeatureFlagsService** - Main service class managing GraphQL client lifecycle and authentication
- **featureFlagsService** - Exported singleton instance for application-wide use
- **FeatureFlagResponse** - Interface for a single flag result (`name`, `enabled`)
- **FEATURE_FLAGS_QUERY** - Pre-defined `feFeatureFlags(names)` query; every requested name is echoed back with its effective value (unknown flags resolve to `enabled: false`)

## Usage Example

```typescript
import { featureFlagsService } from './featureFlagsService'

// Fetch specific flags (pass the full list the app cares about)
const flags = await featureFlagsService.fetchFeatureFlags(['thinking', 'batch-approval'])
if (flags) {
  for (const flag of flags) {
    console.log(`${flag.name}: ${flag.enabled}`)
  }
} else {
  console.log('Fetch failed - fall back to default flag values')
}

// Omitting names returns every flag configured on the server
const allFlags = await featureFlagsService.fetchFeatureFlags()
```

The service automatically handles authentication via `tokenService` (waiting for the token with `ensureTokenReady`), reuses a cached GraphQL client with refreshed headers per call, and returns `null` on any error so callers (e.g. `useFeatureFlagsQuery`) can fall back to default flag values.
